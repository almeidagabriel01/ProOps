<objective>
Strip `?redirect=` consumption from the post-login flow and replace the session-expired subtitle with a warning toast.

Purpose: Eliminate the open-redirect surface and the redirect-loop bugs caused by stale or manipulated `redirect=` URLs (LOGIN-01). After this plan, every authenticated user lands on `/dashboard` (or `/admin` for superadmins) regardless of the URL params, and session-expired notifications fire as a transient toast on the login page instead of a copy swap.

Output:
- `useLoginForm.ts` — `redirectUrl` const removed, `handleRedirectAfterAuth` simplified, toast `useEffect` added, `isPathAllowedForUser` import dropped, `redirectReason` removed from outer redirect useEffect deps, `verifyUrl` and `getGoogleSetupTarget` collateral cleanup.
- `login/page.tsx` — Conditional `session_expired` subtitle removed; subtitle reverts to default copy. `redirectReason` destructure removed from hook return usage if no longer needed at page level.
- `protected-route.tsx` — Bounce URL strips `redirect=`, keeps only `redirect_reason=session_expired`.
- `tests/e2e/seed/data/users.ts` — `SeedUserSuperadminRole` interface + `USER_SUPERADMIN` constant + tenant doc seeded for superadmin in `seedUsers()`.
- `tests/e2e/seed/seed-factory.ts` — `user-superadmin` UID added to `clearAll()` UID list.
</objective>
