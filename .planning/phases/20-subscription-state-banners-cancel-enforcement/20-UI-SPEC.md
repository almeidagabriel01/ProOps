### Banner Color Specifics

**Red banner (past_due):**
- Container: `bg-destructive/10 border-b border-destructive/30`
- Text: `text-destructive` (icon + message)
- CTA button: `variant="destructive" size="sm"`

**Yellow banner (cancelAtPeriodEnd):**
- Container: `bg-yellow-50 dark:bg-yellow-950/90 border-b border-yellow-300 dark:border-yellow-700`
- Text: `text-yellow-800 dark:text-yellow-200`
- CTA button: `variant="outline" size="sm"` with `border-yellow-500 text-yellow-700 hover:bg-yellow-100 dark:border-yellow-500 dark:text-yellow-300` — disabled state for stub

**Banner layout:** Full-width (`w-full`), no border-radius, inserted as a horizontal bar between `<Header />` and `<SubscriptionGuard>` in `ProtectedAppShell`. NOT the floating card pattern from the existing `subscription-guard.tsx` warning — that is a different positioning approach.

## Interaction Contracts

### Banner Dismissal
- Neither banner can be dismissed — no close button, no dismiss state.
- Session-only pattern: if a future enhancement adds session dismiss, use `useState` (no `localStorage`). Not in scope for this phase.
- Banners reappear on page reload — this is intentional.

Source: CONTEXT.md locked decision ("Cannot be permanently dismissed")

### Banner Priority Logic
```
if (subscriptionStatus === 'past_due')  → show red banner only
else if (cancelAtPeriodEnd === true)    → show yellow banner only
else                                    → show neither
```
Both conditions true simultaneously: red banner wins, yellow is suppressed.

Source: CONTEXT.md locked decision

### Yellow Banner "Reativar assinatura" Button
- Rendered in this phase as `disabled={true}`.
- Tooltip on hover: "Disponível em breve".
- `onCta` prop receives an empty function `() => {}`.
- Phase 21 wires up the actual endpoint and removes the disabled state.
- Do NOT omit the button — its presence communicates the recovery path to the user.

### Red Banner "Atualizar pagamento" Button
- Enabled at all times while `past_due` is true.
- Clicking sets local `isRedirecting = true` and calls `StripeService.createPortalSession({ userId: user.id })`.
- On success: `window.location.href = result.url` (redirects to Stripe portal).
- Return URL: `window.location.href` (current page) — same as existing `handleManageBilling` pattern.
- On error: reset `isRedirecting`, banner remains. No toast, no error message to user (matches existing `subscription-guard.tsx` error handling at line 155).
- Button label during loading: "Abrindo..." — matches existing `subscription-guard.tsx` pattern.

### Past_due Cancel AlertDialog
- Opens when `past_due` tenant clicks "Cancelar assinatura" in `MySubscriptionTab.tsx`.
- Uses a separate `open` state from the active-path dialog (or same state with variant — executor's choice, but the dialog content must differ).
- The copy in this dialog is the ONLY visible signal that cancellation is immediate — no other UI indicates this.
- Confirm button (`AlertDialogAction`) triggers `handleConfirmCancelSubscription()` which calls the cancel endpoint.
- Cancel button (`AlertDialogCancel`) dismisses with no action.
