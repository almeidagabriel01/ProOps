<objective>
Resolve the Checkout Pro payment confirmation bug (MPWH-03) and persist MercadoPago fee data on confirmed transactions (MPWH-04).

Today, payments created via Checkout Pro store the `preferenceId` as `payment_attempts.mpPaymentId`. When MP fires `payment.created`/`payment.updated`, `data.id` is the actual payment ID — never the preferenceId — so the direct `where("mpPaymentId", "==", dataId)` lookup always returns empty and the webhook silently logs "no payment attempt found" and exits. Payments are silently lost.

This plan adds the fallback: when the direct lookup is empty, call `GET /v1/payments/{dataId}` using a platform access token (we don't know the tenant yet), parse the `external_reference` field in the MP-confirmed format `${transactionId}:${attemptId}`, and look up `payment_attempts/{attemptId}` directly. Additionally, fee fields already returned in that same MP API call (`transaction_amount` for gross, `transaction_details.net_received_amount` for net) are persisted on the transaction so Phase 24 can display the Bruto/Taxa/Líquido breakdown.

Purpose: MPWH-03 (fallback resolution) + MPWH-04 (fee persistence). Both modify `handlePaymentEvent` in the same file; both are independent of Plan 01's entry-point hardening but build on top of it (Plan 01's idempotency gate already protects this expanded code path from duplicate execution).

Output: Modified `mercadopagoWebhook.ts` with extended `MpPaymentResponse` interface, `parseExternalReference` helper, fallback resolver branch in `handlePaymentEvent`, and fee field writes inside the existing `db.runTransaction()`. New env var documented in `.env.example`. Test coverage in `mercadopagoWebhook.test.ts` includes `jest.mock('axios')` failure-mode unit tests satisfying CLAUDE.md Bug Fix Policy (tests that fail without the fix and pass with it).
</objective>

     * Parses MP payment external_reference field.
     * Format guaranteed by transaction-payment.service.ts (lines 311, 428, 538, 748):
     *   `${transactionId}:${attemptId}`
     * Returns null if the reference is missing, malformed, or either segment is empty.

    export function parseExternalReference(
      ref: string | undefined | null,
    ): { transactionId: string; attemptId: string } | null {
      if (!ref || typeof ref !== "string") return null;
      const parts = ref.split(":");
      if (parts.length !== 2) return null;
      const transactionId = parts[0].trim();
      const attemptId = parts[1].trim();
      if (!transactionId || !attemptId) return null;
      return { transactionId, attemptId };
    }

    export async function handlePaymentEvent(dataId: string): Promise<void> {
      // 1. PRIMARY: direct lookup by mpPaymentId (existing behavior — unchanged)
      const attemptsSnap = await db
        .collection(PAYMENT_ATTEMPTS_COLLECTION)
        .where("mpPaymentId", "==", dataId)
        .limit(1)
        .get();

      let attemptDoc: FirebaseFirestore.QueryDocumentSnapshot | FirebaseFirestore.DocumentSnapshot;
      let attempt: { transactionId: string; tenantId: string; status: string };
      let lookupResult: "primary" | "fallback_resolved";

      if (!attemptsSnap.empty) {
        attemptDoc = attemptsSnap.docs[0];
        attempt = attemptDoc.data() as { transactionId: string; tenantId: string; status: string };
        lookupResult = "primary";
      } else {
        // 2. FALLBACK: Checkout Pro path — preferenceId stored in mpPaymentId, real payment.id differs.
        //    Call MP API with PLATFORM token (we don't know tenantId yet), parse external_reference,
        //    look up payment_attempts.doc(attemptId) directly.
        const platformToken = process.env.MERCADOPAGO_PLATFORM_ACCESS_TOKEN;
        if (!platformToken) {
          logger.warn("MP webhook: fallback unavailable — MERCADOPAGO_PLATFORM_ACCESS_TOKEN not configured", {
            mpPaymentId: dataId,
            lookup_result: "not_found",
          });
          return; // outer handler in onRequest will finalize webhookEvents → "done", HTTP 200
        }

        let mpFallbackResponse: MpPaymentResponse;
        try {
          const resp = await axios.get<MpPaymentResponse>(
            `${MP_API_BASE}/v1/payments/${dataId}`,
            { headers: { Authorization: `Bearer ${platformToken}` } },
          );
          mpFallbackResponse = resp.data;
        } catch (err) {
          logger.warn("MP webhook: fallback MP API call failed", {
            mpPaymentId: dataId,
            error: err instanceof Error ? err.message : String(err),
            lookup_result: "not_found",
          });
          return; // expected unresolvable — outer handler responds 200
        }

        const parsed = parseExternalReference(mpFallbackResponse.external_reference);
        if (!parsed) {
          logger.warn("MP webhook: fallback failed — external_reference missing or malformed", {
            mpPaymentId: dataId,
            externalReference: mpFallbackResponse.external_reference ?? null,
            action: "payment",
            lookup_result: "not_found",
          });
          return;
        }

        const attemptRef = db.collection(PAYMENT_ATTEMPTS_COLLECTION).doc(parsed.attemptId);
        const attemptSnap = await attemptRef.get();
        if (!attemptSnap.exists) {
          logger.warn("MP webhook: fallback failed — payment_attempts doc not found", {
            mpPaymentId: dataId,
            attemptId: parsed.attemptId,
            transactionId: parsed.transactionId,
            externalReference: mpFallbackResponse.external_reference,
            lookup_result: "not_found",
          });
          return;
        }

        attemptDoc = attemptSnap;
        attempt = attemptSnap.data() as { transactionId: string; tenantId: string; status: string };
        lookupResult = "fallback_resolved";

        logger.info("MP webhook: payment attempt resolved via fallback", {
          mpPaymentId: dataId,
          attemptId: parsed.attemptId,
          transactionId: parsed.transactionId,
          tenantId: attempt.tenantId,
          lookup_result: lookupResult,
        });
      }

      const { transactionId, tenantId } = attempt;

      // 3. EXISTING BEHAVIOR continues here unchanged — MercadoPagoService.getMercadoPagoData(tenantId),
      //    fetch payment with tenant token, the rest of the existing handlePaymentEvent body.
      // ... (the rest of the function from current line ~104 onward stays identical) ...
    }

| T-23-07 | Elevation of Privilege | Platform access token misused as tenant credential | mitigate | Token used ONLY inside the fallback branch to fetch the MP payment by ID; after `external_reference` is parsed, code immediately switches to the tenant's OAuth token via `MercadoPagoService.getMercadoPagoData(tenantId)` for all subsequent operations. | T-23-08 | Tampering | Attacker forges `external_reference` to point at another tenant's payment_attempts | accept | The fallback `parseExternalReference` returns `{transactionId, attemptId}`; the direct lookup at `payment_attempts.doc(attemptId)` returns the attempt's stored `tenantId`.
