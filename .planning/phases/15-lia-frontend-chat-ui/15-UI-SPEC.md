## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip inline padding |
| sm | 8px | Bubble padding-x, input field inner gaps |
| md | 16px | Panel content padding, message vertical gap |
| lg | 24px | Panel header vertical padding, section gaps |
| xl | 32px | Panel-to-viewport edge clearance |
| 2xl | 48px | Trigger button bottom offset (`bottom-6` = 24px; trigger stack top offset) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Trigger button: `bottom-6 right-6` = 24px fixed offset (Tailwind default `bottom-6`)
- Touch target minimum for trigger button and send button: 44px × 44px
- Panel width: 420px (fixed, from `decisions` — not on the 8-point scale, frozen by upstream decision)
- Panel slide transition: `translate-x-full` → `translate-x-0`, 300ms ease-in-out

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body (chat text) | 14px | 400 (regular) | 1.6 |
| Label (header, badge, chips) | 12px | 600 (semibold) | 1.4 |
| Heading (panel title "Lia") | 16px | 600 (semibold) | 1.2 |
| Display | Not used in this phase | — | — |

Notes:
- Markdown prose (rendered on `done` SSE event): `prose prose-sm dark:prose-invert` from Tailwind Typography. Body text stays 14px within prose.
- Streaming raw text: 14px regular, same as body — no markdown parsing during stream.
- Typing indicator ("Lia está digitando..."): 12px, muted-foreground, weight 400.
- Usage badge: 12px semibold — uses Badge component variant.
- Error indicator inline badge ("Resposta interrompida"): 11px semibold, destructive variant.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` oklch(0.96 0.005 265) light / oklch(0.145 0 0) dark | Panel body, message area background |
| Secondary (30%) | `var(--card)` oklch(0.99 0.01 250) light / oklch(0.205 0 0) dark | Panel shell (`<aside>`), user message bubble background, header surface |
| Accent (10%) | `var(--primary)` oklch(0.35 0.15 250) light / oklch(0.6 0.15 250) dark | Specific elements listed below |
| Destructive | `var(--destructive)` oklch(0.577 0.245 27.325) light / oklch(0.5 0.2 25) dark | "Resposta interrompida" badge, delete confirm dialog button |

Accent reserved for:
1. Trigger button background (filled circle button)
2. Lia avatar initials background (uses `tenant.primaryColor` from TenantContext, falls back to `var(--primary)`)
3. Send button background
4. Active quick-action chip ring/border
5. Usage badge when ≥ 80%: switches from `var(--primary)/10` tint to `var(--destructive)/10` warning variant
6. Pulse dot notification on trigger button

Secondary color rules:
- Lia message bubbles: `var(--muted)` background (`oklch(0.95 0.015 250)` light)
- User message bubbles: `var(--card)` background with `var(--border)` border
- Panel header: `var(--card)` with bottom `var(--border)` separator
- Input bar: `var(--card)` background, `var(--input)` for textarea field

Dark mode: all colors resolve automatically via CSS custom properties. No hardcoded hex values in component classes.

## Interaction Contracts

### LiaTriggerButton
- Renders only when `planId !== 'free'`
- Position: `fixed bottom-6 right-6 z-50`
- Size: 52px × 52px (exceeds 44px minimum touch target)
- Default state: Sparkle icon (`lucide-react` `Sparkles`) on primary background, white icon
- Open state: × icon (`lucide-react` `X`), same size and color
- Icon transition: `transition-all duration-200 ease-in-out` on the icon swap (CSS opacity cross-fade or rotate)
- Notification dot: `absolute top-0 right-0 h-2.5 w-2.5 rounded-full bg-primary animate-pulse` — visible when panel is closed and `done` SSE event received. Hidden when panel opens.
- Hover: `hover:scale-105 transition-transform`

### LiaPanel
- `<aside>` element, `fixed inset-y-0 right-0 w-[420px] z-40 flex flex-col`
- Surface: `bg-card border-l border-border shadow-xl`
- Open/close: `translate-x-0` (open) / `translate-x-full` (closed), `transition-transform duration-300 ease-in-out`
- Rendered in `ProtectedAppShell` as sibling to `<main>`, outside scrollable content
- Not rendered at all for `planId === 'free'`

### LiaPanel Header
- Layout: `flex items-center gap-3 px-4 py-3 border-b border-border`
- Left: Avatar (32px) with initials "LI", background = `tenant.primaryColor` (inline style)
- Center: "Lia" (16px semibold) + "Assistente ProOps" (12px muted-foreground)
- Right: `LiaUsageBadge` + dropdown/icon button (three-dot `MoreHorizontal` icon) with "Novo Chat" action

### LiaChatWindow
- `flex-1 overflow-y-auto px-4 py-4 flex flex-col gap-3`
- Scroll to bottom on new message (`useEffect` + `scrollIntoView`)
- Typing indicator: three-dot bounce — three 8px circles with CSS staggered `animate-bounce` (delay 0ms, 150ms, 300ms), color `var(--muted-foreground)`

### LiaMessageBubble
- Lia message: `max-w-[85%] self-start rounded-2xl rounded-tl-sm px-3 py-2 bg-muted text-foreground text-sm leading-relaxed`
- User message: `max-w-[85%] self-end rounded-2xl rounded-tr-sm px-3 py-2 bg-card border border-border text-foreground text-sm`
- Streaming state: render raw text token by token as plain text
- Post-stream (`done` event): replace with `<ReactMarkdown className="prose prose-sm dark:prose-invert">` render
- Error badge: `<span className="inline-flex items-center gap-1 ml-2 px-1.5 py-0.5 text-[11px] font-semibold rounded bg-destructive/10 text-destructive">Resposta interrompida</span>` appended inline after partial content

### LiaInputBar
- `flex items-end gap-2 px-4 py-3 border-t border-border bg-card`
- Textarea: auto-grow, max 5 lines, `resize-none`, `text-sm`
- Send button: 44px × 44px, `SendHorizontal` lucide icon, primary background
- Disabled state: textarea and send button disabled when `messagesUsed >= messagesLimit`
- Disabled tooltip (Radix Tooltip): shows reset date — "Limite atingido. Renova em {date}."
- `Cmd+Enter` / `Ctrl+Enter` submits message (keyboard shortcut)

### LiaToolConfirmDialog
- Uses existing `dialog.tsx` (Radix Dialog)
- Title: "Confirmar ação"
- Body: Lia's natural language description of the action
- Buttons: "Cancelar" (outline) + "Confirmar" (destructive variant) — both 44px min height
- On "Confirmar": hook resends with `confirmed: true`
- On "Cancelar": sends cancellation signal, Lia acknowledges in chat

### LiaToolResultCard
- Collapsed by default: shows tool name + icon + 1-line result summary
- Expand/collapse: Radix Collapsible, chevron icon rotates 180° on open
- Full result: JSON or human-readable content inside `<pre>` or formatted list
- Background: `bg-muted/50 rounded-xl border border-border p-3`

### LiaUsageBadge
- Format: "{messagesUsed} / {messagesLimit}"
- Below 80%: `variant="secondary"` (neutral)
- At or above 80%: `variant="warning"` (yellow)
- At limit: `variant="destructive"` (red)

### Quick-Action Chips (Starter plan)
- 2-3 chips rendered below greeting bubble
- Chip style: `inline-flex items-center gap-1.5 px-3 py-1.5 rounded-full border border-border bg-card text-sm text-foreground hover:bg-muted transition-colors cursor-pointer`
- Click sends the chip text as a user message

## Accessibility Contract

- All interactive elements: min 44px touch target
- Trigger button: `aria-label` changes with open state
- Dialog: full focus trap via Radix Dialog (already implemented)
- Chat window: `role="log"` `aria-live="polite"` for streaming message updates
- Typing indicator: `aria-label="Lia está digitando"` on the bounce container, `aria-live="polite"`
- Input textarea: `aria-label="Mensagem para Lia"`
- Usage badge: `aria-label="{messagesUsed} de {messagesLimit} mensagens usadas"`

## Animation Contract

| Element | Animation | Duration | Easing |
|---------|-----------|----------|--------|
| Panel slide in/out | `translate-x-0` ↔ `translate-x-full` | 300ms | `ease-in-out` |
| Trigger icon swap (chat ↔ ×) | opacity cross-fade + `rotate-90` transform | 200ms | `ease-in-out` |
| Trigger notification dot | `animate-pulse` (Tailwind built-in) | continuous | — |
| Typing indicator dots | `animate-bounce` × 3, staggered 150ms | continuous | — |
| Quick-action chips appear | `animate-in fade-in slide-in-from-bottom-2` | 200ms | `ease-out` |
| Tool result card chevron | rotate 0° ↔ 180° | 200ms | `ease-in-out` |

All animations use existing Tailwind v4 utilities and `tw-animate-css`. No new animation dependencies.
