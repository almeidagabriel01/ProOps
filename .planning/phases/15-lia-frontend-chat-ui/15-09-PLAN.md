<objective>
Fix dashboard regression: TypeError when rendering proposals with undefined clientName.

Purpose: `p.clientName.substring(0, 2)` at line 157 of `recent-lists.tsx` throws when `clientName` is undefined. The `mapProposalDoc` function in `proposal-service.ts` uses `...data` spread without defaulting `clientName`, so runtime value is `undefined` despite the TypeScript type declaring `clientName: string`.

Output: Null-safe dashboard rendering + defensive default in the mapping function.
</objective>

function mapProposalDoc(d: QueryDocumentSnapshot<DocumentData>): Proposal {

Add an explicit `clientName` field AFTER the `...data` spread so it overrides the spread value with a safe default:

clientName: (data.clientName as string) || "",
